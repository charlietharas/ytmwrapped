# Project Context: YouTube Music Wrapped Analyzer

This file describes YTMWrapped, a client-side, in-browser "YouTube Music Wrapped" application. It allows users to upload their Google Takeout history (at least one `watch-history.json` file) and visualize their listening habits without their data ever leaving their browser.

Use React with consistent rendering structure for frontend, grab data from a Python analysis script
- The script performs an initial analysis step to generate one or more Pandas dataframes (this step can take a while and do the heavy lifting), then easily accesses these dataframes to quickly apply filters (using under-the-hood tricks to optimize performance, e.g. caching aggregated data or storing them in separate dataframes)

We refer to a unique song by its YouTube ID (this is the canonical ID/primary key)
- This ID is parsed from the `titleUrl` field of each history entry (e.g., `...watch?v=VIDEO_ID`).
- All aggregations (top songs, repeats, weekly charts) are grouped by `video_id`.
- For display purposes, the `video_id` is mapped back to a human-readable `Artist - Title` string before being sent to the frontend

## Frontend flow

1. Header always present, has title and Preferences kebab on the top right that pops sectioned mini menu:
    - Options
        - Show privacy disclaimer and instructions toggle (enabled by default)
        - Cache last processed CSV toggle (enabled by default)
            - Uses LocalStorage to cache the dataframe generated by the initial analysis stage (caches only most recent)
            - Deletes cache on toggle off (with a confirm popup if cached CSV is present)
        - Cache API requests for songs toggle (enabled by default)
            - Uses LocalStorage to cache a hashmap of song ID → artist, duration, genre, release year
            - This hashmap persists and is updated across multiple flows
            - Deletes cache on toggle off (with a confirm popup if cached hashmap is present)
    - Clear data
        - Delete my Google API key button
        - Delete durations data button
        - Delete MusicBrainz data button
        - Reset button
            - Deletes absolutely everything from LocalStorage and restarts
    - Export
        - Export processed CSV button (exports dataframe as CSV, greyed out when dataframe does not exist)
        - Export report as PDF button (exact specifics TBD, greyed out for now)
2. On load, first show privacy disclaimer and instructions with “Next” button (if preference not cached as no)
    - Two separate sections: disclaimer and instructions, each toggleable
    - “Don’t show this again” toggle beside “Next” caches preference to local storage (toggle disabled by default)
    - Load Python environment in the background in this screen, if screen is disabled or user moves past, have loading indicator while it loads before moving to next stage
3. Upload box stage
    - Drag and drop box to upload several JSON files which are automerged (while avoiding duplicates) OR to upload generated CSV report
        - Error message if processing script fails
    - If a cached CSV is detected on load, upload is split vertically with another smaller box to just “load previous report”
4. Once files are uploaded/previous report loaded, show loading indicator while initial analysis performs
    - Loading indicator should be centered and blur page behind it when any analysis is being performed (like this initial one, or when applying a filter)
    - This stage should result in the initial dataframe being generated (should be near-instant if using cached or uploaded CSV)
    - Then after the dataframe is generated, the main screen should be active:

## Main screen

### Filters at the top (sticky, below the header)

- Each card with a chart/table has a zoom toggle that displays when filters are enabled
    - When filters are enabled, charts are stacked with items matching the filter in red, and not matching in grey
        - Tables highlight items matching the filter in red, but only considering song, artist, duration, genre, release year filters
            - Artists in table and chart are highlighted if they match the artist of a selected song in filter, or if any of their songs match the genre or release year selected in filter (no effect for duration)
    - When a card zoom is toggled, charts display only the items matching the filter in red instead of all items
        - Tables will show only the highlighted items
- Each filter has a clear button to reset it, and there is a clear all button on the right to reset all filters

Filter list:

- Start date; end date boxes
    - Does not count as a filter being applied when start/end matches the start/end of uploaded data range (and filter shows as empty)
    - Enable click and drag on Timeline chart to filter
    - Enable just manually inputting the date into boxes (reset to empty if outside range)
- Song (OR multiselect)
    - Enable click on any song reference (in a table or chart) to filter
    - Enable clicking on the filter to search through all songs and multiselect from scrollable capped display height dropdown
    - For song filter, we should pop already selected songs to the top of this dropdown
- Artist (OR multiselect)
    - Enable click on any artist reference (in a table or chart) to filter
    - Enable clicking on the filter to search through all artists and multiselect from scrollable capped display height dropdown
    - For artist filter, we should pop already selected artists to the top of this dropdown
- Hour of day (OR multiselect)
    - Enable click on hours chart to filter
    - Enable clicking on the filter to multiselect from scrollable capped display height dropdown
- Day of week (OR multiselect)
    - Enable click on weekdays chart to filter
    - Enable clicking on the filter to multiselect weekdays from uncapped dropdown
- Month of year (OR multiselect)
    - Enable click on months chart to filter
    - Enable clicking on the filter to multiselect months from uncapped dropdown
- Year (OR multiselect)
    - Enable click on years chart to filter
    - Enable clicking on the filter to multiselect years from uncapped dropdown
- Min duration; max duration (shown as slider with labeled ends)
    - Greyed out when Google API data not loaded
    - Enable click and drag on durations chart to filter
    - Enable moving slider ends to filter
- Genre (OR multiselect)
    - Greyed out when MusicBrainz data not yet loaded
    - Enable click on artist chart to filter
    - Enable clicking on the filter to multiselect genres from scrollable capped display dropdown
    - Scrollable capped display dropdown has option that persists at top to include songs with no detected genre (enabled by default)
- Earliest release year; latest release year boxes
    - Greyed out when MusicBrainz data not yet loaded
    - Enable click and drag on release year chart to filter
    - Enable manual input to boxes (reset to empty if outside range)
- Include missing release year (toggle, enabled by default, untoggleable when MusicBrainz data not yet loaded)
- One instance of a song is a “listen” (use this as the quantity counter everywhere)

### Cards

The main pane is broken into various “cards” which have a fixed size, layed out into a grid, and are internally scrollable if need be

Card list:

- Timeline chart card (2x width): histogram of songs by day with X axis formatted by year/month (e.g. 2021 Feb Mar …)
    - This card (and many others) has a “duration toggle”: if durations data is loaded, show card toggle to switch quantity from plays to duration (in this case duration per day)
- API cards
    - Google/YouTube API integration card
        - Toggleable instructions to generate a YouTube API key
        - Input box for YouTube API key
        - Button to fetch durations for all songs, does so in background with progress showing on the button
            - Only populate durations data when job is complete
        - Button to clear durations for all songs (pop confirm, shows “cancel” when job is in progress)
    - MusicBrainz integration
        - Button to fetch genre and year information for all songs, does so in background with progress showing on the button
            - Only populate MusicBrainz data when job is complete
        - Button to clear MusicBrainz data for all songs (pop confirm, show “cancel” when job is in progress)
- Key Statistics card (populated with text)
    - Total plays, unique song count, unique artist count
    - Mean, median, max counts of replays per unique song
    - If duration data is loaded: total duration, mean, median, max, min duration per song and per artist
    - If filters are applied, split card vertically to show all statistics with filters applied
- Temporal section
    - Songs per hour of day circular bar chart card (include hours with no plays)
        - Duration toggle
    - Songs per day of week bar chart card
        - Duration toggle
    - Songs per month of year bar chart card
        - Duration toggle
    - Songs per year bar chart card
        - Duration toggle
- Cumulative section
    - Top 20 artists column chart card
        - Duration toggle
    - Top 20 songs column chart card
        - Duration toggle
    - All artists table
        - Searchbar
        - Duration toggle
        - Note that each row in a table displays a persistent rank, where rank is sorted by total plays/total duration
    - All songs table
        - Searchbar
        - Duration toggle
- Periodic section
    - Top songs per week table
        - Dropdown and arrows to select week
        - Duration toggle
        - Searchbar
    - Top songs per month table
        - Dropdown and arrows to select month
        - Duration toggle
        - Searchbar
    - Top artists per week table
        - Dropdown and arrows to select week
        - Duration toggle
        - Searchbar
    - Top artists per month table
        - Dropdown and arrows to select month

        - Searchbar
- MusicBrainz section
    - Genres

            - Duration toggle
